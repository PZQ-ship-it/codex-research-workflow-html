协作契约先于任务执行

每个项目都应有一个稳定入口，告诉 Codex 项目目标、事实源、术语边界、执行优先级和禁止夸大的地方。

最小协作契约

  • 项目目标：一句话说明当前工作服务什么交付物。
  • 贡献主线：把贡献拆成有限维度，避免每次改稿重塑叙事。
  • 优先事实源：列出机器可读 manifest、hub、术语映射和审计报告。
  • 工作顺序：规定先读什么，再改什么，最后如何验收。
  • 边界：明确哪些说法不能写，哪些数据不能外推。

推荐提示词骨架

你进入本项目后，请先阅读协作入口。
本项目当前目标是：[交付目标]。
优先事实源为：[manifest / hub / term mapping / reports]。
若涉及论文更新，请先判断影响范围，再做一次性长更新。
若涉及实验或图表，请保留可恢复日志、manifest 和验收证据。
禁止：[不可夸大的贡献、不可使用的数据、不可直接生成的事实]。

复用经验

把 Codex 从“回答问题的人”变成“进入项目后遵守规则的协作者”。这会减少上下文漂移，也能让后续代码、图表、论文改动围绕同一组事实收敛。
