Human Workflow Recipes 可直接复制给 Codex 的任务提示
首页 手册 工具地图 触发句式

从“请帮我做”到“请按证据闭环做”

下面这些 recipes 是给人的 prompt 模板。复制后替换路径、目标和验收标准即可。

目录

开发任务

目标：[一句话说明功能/bug]
请先探索相关入口、测试和现有模式，再实施最小改动。
要求：
- 不改无关文件；
- 用补丁方式编辑；
- 改完运行相关测试；
- 最后说明变更文件、验证命令和剩余风险。

论文事实同步

目标：根据 [实验/数据/事实源路径] 更新论文相关表述。
请先读项目协作入口、claim manifest、term mapping 和受影响章节。
先更新事实源，再一次性同步正文、图表说明、摘要/结论中受影响的表述。
最后搜索旧术语、旧数字和过度声称，并给出检查证据。

实验跑批

目标：运行 [benchmark name]。
请把 runner 设计成可恢复：
- 组合键为 [model, profile_id, mode]；
- append 写结果；
- status/manifest/logs 全部落盘；
- API 错误重试，输入错误单独标记；
- 汇总前检查缺失和失败。
完成后生成论文可引用 summary，但不要夸大 partial run。

作图与论文同步

目标：从 [CSV/result path] 生成 [figure id]。
请更新或创建绘图脚本：
- 固定输入、输出、LaTeX 图目录；
- 中文字体、标签映射、方法顺序和色板集中配置；
- 输出 PNG + PDF；
- 写 summary；
- 同步到 LaTeX 后渲染关键页截图验收。

LaTeX / PDF 终稿审阅

目标：检查 [PDF 或 LaTeX root] 是否可提交。
请执行终稿审阅：
- 编译或确认最新 PDF；
- 检查 error、引用、图片、overfull warning；
- 抽取文本查术语、指标、旧口径；
- 渲染封面、摘要、目录、关键图表和结论页；
- 把自动检查和人工确认项分开列出。

HTML 笔记/文档

目标：把 [对话/流程/决策] 整理成 HTML 文档。
要求：
- 不使用 Markdown；
- 有元数据、导航、内部链接和 references；
- 样式适合阅读和打印；
- 用 Playwright 做桌面/移动截图验收；
- 链接检查通过。

把工作流沉淀成 skill

这个任务重复出现且容易遗漏步骤。请使用 $skill-creator 创建一个 skill。
要求：
- description 写清楚触发场景和边界；
- SKILL.md 保持精简；
- 详细清单放 references；
- 可复用模板放 assets；
- 运行 quick_validate.py；
- 给出安装/重启说明。
