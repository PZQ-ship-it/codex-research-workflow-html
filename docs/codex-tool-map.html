工具矩阵

工具/能力 何时使用 Prompt 触发方式 要求的证据 常见误用 你的项目例子
Shell / PowerShell 读文件、搜索、跑测试、启动本地服务、检查 Git 状态。 “请先用 rg 搜索相关文件，再决定改动范围。” 关键命令、测试结果、失败日志摘要。 让模型猜仓库结构；用 shell 拼复杂写文件。 搜索论文事实源、跑 LaTeX 编译、检查实验 CSV。
apply_patch 小范围、可审查的代码或文档修改。 “请用补丁方式做最小变更，避免无关重构。” 变更文件列表、核心 diff 说明、验证命令。 用脚本重写大文件导致不可读 diff。 修改图表脚本、HTML 文档、LaTeX 小段落。
Web search / open 最新信息、官方文档、法律/标准/价格/API 变化。 “请联网查官方来源，并在答案里给链接。” 来源链接、发布日期或访问到的官方页面。 用旧记忆回答动态问题；引用非官方二手资料。 查 OpenAI Codex tools、模型/文档更新。
Specialized web data 金融、天气、体育、时间等结构化实时查询。 “请查当前价格/天气/赛程，不要凭记忆。” 时间戳、地点/代码、查询结果摘要。 把实时值写成长期稳定结论。 论文一般少用；适合外部背景材料快速核对。
MCP 需要连接外部系统：Figma、Notion、OpenAI Docs、GitHub、Sentry、浏览器。 “请用 MCP 读取 Figma/Notion/官方 docs 上下文。” MCP 资源、节点、页面或工具调用结果。 没有连接 MCP 却假装读到了外部系统。 Figma 设计转代码、Notion 资料研究、官方 docs 查询。
Skills 重复流程、有固定规则、容易遗漏步骤。 显式：$pdf、$skill-creator；隐式：描述任务让 Codex 匹配 skill。 skill 名称、使用的资源、执行后的验收结果。 把所有任务都做成 skill，导致灵活度下降。 PDF 审阅、图表流水线、HTML 笔记、可恢复跑批。
Subagents 多个独立问题可并行探索或实现，且结果可汇总。 “请 spawn 一个 agent 查测试，一个 agent 查图表，一个 agent 查论文引用。” 每个 subagent 的结论、文件路径、剩余风险。 把关键阻塞任务交出去后主线程空等；并行写同一文件。 大型代码审查、论文多章节一致性检查。
Playwright / browser 网页、HTML 文档、前端工具、图表页面需要真实浏览器验收。 “请用 Playwright 打开页面，截图并检查 console。” 截图路径、console 错误数、视口尺寸、布局检查。 只看源码不看浏览器；只截桌面不看移动端。 HTML workflow kit 的首页和模板页验收。
PDF skill / PDF 工具 论文、报告、版式、页面渲染、文本抽取。 “请用 PDF 工作流渲染关键页，检查图表和文字。” PDF 路径、页码截图、抽取文本、编译 warning。 只读 LaTeX 源码，不看最终 PDF。 毕业论文终稿、开题/中期报告、图表插入检查。
Image generation 需要位图插画、纹理、mockup、透明背景素材。 “请用 imagegen 生成一张可用作……的位图素材。” 生成文件路径、尺寸、用途说明。 用它做应由 SVG/CSS/图表脚本完成的结构图。 若做展示页背景图可用；论文图表优先脚本生成。
Screenshot 需要系统级截图、窗口截图或工具无法捕捉的画面。 “请截取当前桌面/某窗口/某区域。” 截图路径或可视检查结论。 能用 Playwright/PDF 渲染时仍用系统截图，降低可复现性。 IDE 状态、系统窗口、非网页软件界面留证。
Figma skills 设计稿读取、Figma 到代码、代码/描述写入 Figma。 “请用 Figma MCP 获取 node 的设计上下文和截图。” node id、截图、设计 token、实现对比。 跳过截图只看结构；硬编码设计 token。 以后把 HTML playbook 视觉系统沉淀到 Figma。
Notion skills 会议、决策、研究材料需要进入 Notion 知识库。 “请把这段对话整理成 Notion decision log / how-to。” 页面标题、数据库、链接和关系。 没有连接 Notion MCP 却声称已写入。 把论文协作经验沉淀为团队 wiki。
