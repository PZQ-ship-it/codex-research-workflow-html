不要让论文正文成为唯一事实源

正文适合叙述，manifest 适合约束。科研项目越复杂，越需要把声明、指标、术语、实验状态和影响章节拆出来维护。

事实源层级

层级内容作用
Claim manifest论文声明、数值、状态、出处防止实验结果和正文描述不一致。
Term mapping术语、工程标识、中文写法防止一个概念在不同章节有多个名字。
Document hub受影响文档、同步清单支持一次性长更新。
Evidence reports实验摘要、日志、审计样例支撑正文中的结论和图表。

同步流程

  1. 识别本次变更影响哪些声明、指标、术语、图表和章节。
  2. 更新或创建 manifest 和 term mapping。
  3. 按 document hub 同步论文正文、图表说明、摘要、结论和附录。
  4. 做关键词检索，确认旧口径没有残留。
  5. 编译或渲染最终产物，记录检查结果。

防过度声称

  • 把“已覆盖”与“已人工逐条确认”分开写。
  • 把“主实验”和“扩展实验 / pilot / 压力测试”分开写。
  • 把“事实候选来源”和“语言解释来源”分开写。
  • 失败样本和适用边界进入正文或局限性，不藏在日志里。
