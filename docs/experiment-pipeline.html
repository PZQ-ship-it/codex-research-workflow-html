首页实验模板

长任务必须能中断、恢复和审计

当实验涉及多模型、多样本、多模式和外部 API 时，断点续传、分片、重试、status 文件和 manifest 是基本设施。

目录

推荐产物

产物用途
experiment_manifest.json/html记录模型、数据集、模式、指标和恢复策略。
results.csv追加写入结果，用组合键跳过已完成任务。
status.html记录完成数、失败数、待重试项和最近错误。
logs/保存每个分片或 case 的错误和摘要。

执行纪律

  • 结果实时 append，不在最后一次性写出。
  • 启动时读取历史结果，跳过已完成组合键。
  • API 错误使用指数退避，区分限流、连接错误和不可恢复输入错误。
  • 被测模型和模拟器模型分开配置，避免成本失控。
  • 所有随机扩增样本要保存 seed、生成提示和审计字段。

验收问题

  1. 失败后重启是否会跳过已完成任务？
  2. 任意一条论文指标是否能追溯到 CSV 行和 case 日志？
  3. 模型切换是否只影响被测对象，不污染 simulator / evaluator？
  4. 异常样本是否有解释，而不是被静默丢弃？
