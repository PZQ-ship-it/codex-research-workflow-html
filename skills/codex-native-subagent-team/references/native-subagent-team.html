Codex Native Subagent Team Recipe

Use this recipe when the user explicitly authorizes native subagents or parallel delegation. It adapts OMX $team discipline without copying tmux, hooks, or .omx/state.

Lane Selection

LaneUse forAvoid when
explorerSpecific repo questions, file/symbol mapping, implementation discovery.The answer blocks the immediate next local action.
workerBounded edits with clear file ownership and disjoint write scope.Multiple workers would edit the same files or broad behavior.
main threadCritical path, shared-file edits, integration, final verification.The work is independent and can safely run in parallel.

Good Prompt

Use native subagents for these independent lanes.

Main thread:
- Owns updating docs/codex-advanced-config-and-usage.md.
- Will not duplicate the delegated repo inventory.

Explorer A:
- Goal: find all repo-local Codex skills and summarize their trigger scenarios.
- Scope: read-only, skills/ and docs/ only.
- Output: concise table with paths and gaps.

Explorer B:
- Goal: inspect docs for stale OMX migration claims.
- Scope: read-only, docs/*.html and docs/*.md.
- Output: findings with file paths and suggested edits.

Integration:
- Main thread merges findings, edits docs, and runs a final rg sanity check.

Bad Prompt

Spawn several agents to improve the repo.

This is bad because the agents have no ownership boundary, no output contract, and no verification target.

Final Report Checklist

  • Main-thread changes are listed.
  • Each subagent contribution is summarized separately.
  • Verification commands or artifact checks are included.
  • Unrun checks and residual risk are explicit.
