# Formal Review Report Template

Use this template for a full Chinese thesis or manuscript review. Adapt headings to the user's context.

## 封面信息

```
论文评审意见
《论文题目》

评审对象：本科/硕士/博士/投稿论文/课程论文/内部草稿
作者：如已知
评审口径：如“非正式修改评审”“答辩前风险评审”“投稿前技术评审”
形成日期：YYYY 年 M 月 D 日

初评结论：建议通过/大修后通过/需重大修改/暂不建议提交；一句话说明原因。
说明：本报告用于论文修改与答辩准备参考，不替代正式评审意见。
```

## 一、总体评价

写 2-4 段：

- 第一段指出选题价值、真实问题和最有辨识度的贡献。
- 第二段概括完成度：问题定义、方法、系统、数据、实验、写作等。
- 第三段指出最影响评分或录用的关键风险。
- 如果需要，第四段说明总体建议和修改后潜力。

## 二、评分建议

按口径调整维度。每个维度的评语必须具体。

| 实验设计与结果论证 |  |  |
| 写作结构与规范性 |  |  |
| 综合建议 |  |  |

## 三、主要优点

列 3-5 项。每项用“判断 + 证据 + 为什么重要”的结构，不写空泛表扬。

```
1. 问题定义有辨识度
   论文在第 X 章将……建模为……，这比单纯……更能解释……。这一点支撑了全文的研究主张。
```

## 四、必须优先修改的问题

列 4-7 项。每项用：

```
1. 问题标题
   位置/证据：第 X 节/公式 X/表 X/实验 X 显示……
   风险：这会导致……
   建议：建议二选一/补充……/改称……/增加……
```

强问题应当落到具体机制，例如公式反直觉、理论命名不足、数据泄漏、基线缺失、统计显著性不足、外部有效性不足、标题与贡献错位。

## 五、写作与结构修改建议

常见高价值建议：

- 增加贯穿全文的完整案例。
- 合并重复术语，首次定义后用具体动作替代口号。
- 调整标题以贴近真实贡献。
- 增补图表、流程、算法伪代码、数据流或误差案例。
- 把局限性从泛泛表态改成具体威胁与后续验证计划。

## 六、答辩风险点与建议回答

| 可能问题 | 建议回答方向 |
|---|---|
| 为什么不用更简单的 baseline/method？ | 说明简单方案能解决什么、不能解决什么；用论文证据说明本文机制的必要性。 |
| 实验是否能代表真实场景？ | 明确当前实验验证范围，承认外部有效性边界，说明补充计划。 |
| 某理论/算法命名是否严格？ | 如实降低表述或补足形式化，避免夸大。 |
| 如何避免关键风险？ | 指向数据、工具、流程、人工审核、日志或统计证据。 |

## 七、修改优先级清单

| 优先级 | 修改项 | 预期效果 |
|---|---|---|
| P0 | 修正会影响正确性、可答辩性或正式通过的硬伤。 | 消除直接质疑点。 |
| P1 | 补充关键实验、统计检验、案例、图表或定位说明。 | 增强可信度与可读性。 |
| P2 | 扩展外部有效性、用户研究、未来工作或表达优化。 | 提升完整度与上限。 |

## 八、最终评审意见

用 2-3 段收束：

- 明确是否达到当前评审口径。
- 概括最主要优势。
- 概括最影响评分/录用/答辩的风险。
- 给出条件式建议：完成哪些 P0/P1 后可提升到什么水平。
