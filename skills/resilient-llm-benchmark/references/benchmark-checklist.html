Resilient Benchmark Checklist

  1. Define matrix: models, profiles, modes, seeds, metrics.
  2. Define resume key and uniqueness constraints.
  3. Write append-only results and skip completed keys on startup.
  4. Separate tested-agent model from simulator and evaluator models.
  5. Use retry/backoff only for transient failures.
  6. Persist status, manifest, logs, and partial summaries.
  7. Audit failed, skipped, and partial rows before computing paper metrics.

Minimum Result Columns

run_id, model, profile_id, mode, seed, status, metric..., error_type, transcript_path, created_at
