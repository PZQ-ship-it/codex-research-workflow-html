Fact Source Sync Checklist

  1. List changed facts, metrics, terms, figures, and affected claims.
  2. Update manifest-like sources before editing prose.
  3. Map engineering identifiers to paper-facing terms.
  4. Update all affected sections in one pass: abstract, contribution, method, experiments, conclusion, captions, appendices.
  5. Search for old terms, old numbers, stale model names, and unsupported superlatives.
  6. Compile or render the final artifact when layout or citation risk exists.
  7. Record remaining human-only confirmations separately.

Claim Boundary Phrases

  • Use “supports”, “indicates”, or “in this benchmark” when scope is limited.
  • Use “coverage” only for assignment/completeness, not semantic correctness unless verified.
  • Use “pilot” or “extension” when an experiment is not part of the main result table.
