HTML Research Note Rules

  • Use standalone HTML that opens directly from disk.
  • Include title, purpose, status, owner, date, and source links.
  • Use relative links for local navigation.
  • Prefer semantic sections and tables over decorative blocks.
  • Keep styling utilitarian, print-friendly, and readable on narrow screens.
  • Validate important pages with browser screenshots when presentation matters.
  • Do not embed secrets, private raw logs, or bulky generated artifacts.
