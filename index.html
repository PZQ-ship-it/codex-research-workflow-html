Codex Research Workflow HTML Kit 开发、论文、实验、图表和审阅的一套可复用骨架
协作约定 事实源 实验 图表 审阅 AI 使用手册 OMX 迁移

把一次性 AI 辅助，升级成可复用的科研工程流水线

这个仓库存放 HTML-first 笔记、模板和 repo-local Codex skills。核心思想是先固定事实源和边界，再让代码、实验、图表、论文围绕同一套事实源迭代，最后用 PDF 与浏览器级验收收口。

状态starter kit
记录格式HTML notes
Skill 位置skills/
适用场景科研原型 / 毕设 / 系统论文

1. Codex 协作约定

给后续 Codex 会话的项目入门路径、事实边界、术语口径和更新顺序。

2. 事实源与论文同步

用 claim manifest、term mapping、document hub 控制论文事实，不让正文漂移。

3. 可恢复实验流水线

大规模 LLM 跑批中的分片、断点续传、重试、日志和模型切换规范。

4. 图表生成与同步

从 CSV 到论文级 PNG/PDF，再到 LaTeX 图目录和视觉验收。

5. 终稿审阅

把源码检查、编译检查、PDF 文本抽取和关键页截图合成一张提交前清单。

6. Repo-local Skills

科研工作流 skills 与 OMX-inspired Codex skills，可复制到 $CODEX_HOME/skills 后复用。

7. 给人看的 AI 使用手册

解释什么时候该让 Codex 查证、读文件、改代码、截图验收、调用 skill 或派生 subagents。

8. oh-my-codex 到 VS Code 的迁移研究

把 OMX 的 deep interview、planning、completion loop、visual QA 等思想改写成 Windows Codex for VS Code 可用的原生 skills。

模板入口

模板用途
claim-manifest.html记录论文声明、事实源、状态和影响章节。
experiment-manifest.html记录实验配置、分片、指标、产物和恢复策略。
figure-manifest.html记录图表输入、脚本、输出、LaTeX 引用和检查结果。
pdf-review-report.html记录 PDF 终稿编译、页面渲染、术语和图表审查。
decision-log.html记录重要设计决策、备选方案、依据和后续动作。
run-log.html记录长任务执行过程、错误、恢复点和验收证据。

本仓库刻意不存放 API key、原始私有日志或大型生成产物。复制到新项目时，先改事实源和术语边界，再复用流程。
